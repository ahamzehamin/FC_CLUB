## üîß **Custom React Hooks & jQuery Conversions**

### **Why Custom Hooks?**

As a full-stack developer, I realized early on that the original jQuery-heavy template needed a complete architectural overhaul. Instead of just copying jQuery code into React components, I decided to create custom hooks that encapsulate the interactive behaviors in a more React-native way. This approach provides several benefits:

- **Reusability**: Hooks can be shared across multiple components
- **Testability**: Easy to test logic in isolation
- **Performance**: Better integration with React's rendering cycle
- **Maintainability**: Centralized logic that's easier to debug and modify

### **Custom Hooks I Created**

#### **1. useStickyNavigation Hook**
```javascript
// src/hooks/useStickyNavigation.js
import { useEffect, useState } from 'react';

const useStickyNavigation = () => {
  const [isScrolled, setIsScrolled] = useState(false);

  useEffect(() => {
    const handleScroll = () => {
      const scrollPosition = window.scrollY;
      const threshold = 100; // pixels from top

      if (scrollPosition > threshold && !isScrolled) {
        setIsScrolled(true);
      } else if (scrollPosition <= threshold && isScrolled) {
        setIsScrolled(false);
      }
    };

    window.addEventListener('scroll', handleScroll, { passive: true });
    return () => window.removeEventListener('scroll', handleScroll);
  }, [isScrolled]);

  return isScrolled;
```

```javascript
// src/hooks/useCounterUp.js
import { useEffect, useRef, useState } from 'react';

const useCounterUp = (endValue, duration = 2000, trigger = true) => {
  const [count, setCount] = useState(0);
  const countRef = useRef(null);
  const [isVisible, setIsVisible] = useState(false);

  useEffect(() => {
    if (!trigger) return;

    const observer = new IntersectionObserver(
      ([entry]) => {
        if (entry.isIntersecting && !isVisible) {
          setIsVisible(true);
        }
      },
      { threshold: 0.3 }
    );

    if (countRef.current) {
      observer.observe(countRef.current);
    }

    return () => observer.disconnect();
  }, [trigger, isVisible]);

  useEffect(() => {
    if (!isVisible) return;

    let startTime;
    const startValue = 0;

    const animate = (currentTime) => {
      if (!startTime) startTime = currentTime;
      const progress = Math.min((currentTime - startTime) / duration, 1);

      // Easing function for smooth animation
      const easeOutQuart = 1 - Math.pow(1 - progress, 4);
      const currentValue = Math.floor(startValue + (endValue - startValue) * easeOutQuart);

      setCount(currentValue);

      if (progress < 1) {
        requestAnimationFrame(animate);
      }
    };

    requestAnimationFrame(animate);
  }, [isVisible, endValue, duration]);

  return [count, countRef];
};

// Usage in Counter component
```

### **Why This Approach Matters**

From my experience as a full-stack developer, converting jQuery to React isn't just about replacing syntax‚Äîit's about rethinking how interactivity works in a component-based architecture:

1. **Performance**: React's virtual DOM and hooks provide better performance than jQuery DOM manipulation
2. **Maintainability**: Centralized logic in custom hooks makes debugging easier
3. **Reusability**: Hooks can be shared across components without duplication
4. **Testing**: React hooks are much easier to test than jQuery event handlers
5. **Modern Patterns**: Uses contemporary JavaScript features and React best practices

This conversion approach ensures the website maintains its original visual appeal and user experience while gaining the benefits of modern React architecture.

```javascript
import { useState, useEffect } from 'react';

const useFetchData = (endpoint, initialData = []) => {
  const [data, setData] = useState(initialData);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    const fetchData = async () => {
      try {
        setLoading(true);
        const response = await axios.get(endpoint);
        setData(response.data.data || initialData);
        setError(null);
      } catch (err) {
        console.error('Error fetching data:', err);
        setError(err.message);
      } finally {
        setLoading(false);
      }
    };

    if (endpoint) {
      fetchData();
    }
  }, [endpoint]);

  const refetch = async () => {
    // Method to manually refetch data
    if (endpoint) {
      try {
        const response = await axios.get(endpoint);
        setData(response.data.data || initialData);
        setError(null);
      } catch (err) {
        setError(err.message);
      }
    }
  };

  return { data, loading, error, refetch };
};
```
